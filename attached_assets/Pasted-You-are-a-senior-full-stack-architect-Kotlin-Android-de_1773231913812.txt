# ITINERARY GENERATION ALGORITHM

The system must:

1. geocode the city
2. fetch POIs from OpenStreetMap / Overpass
3. score attractions by popularity
4. prioritize must-see landmarks
5. filter by user interests
6. filter by travel profile
7. filter by budget
8. group nearby attractions
9. distribute across days based on rhythm
10. generate logical travel order

Avoid random ordering.

Nearby attractions must be grouped in the same day.
